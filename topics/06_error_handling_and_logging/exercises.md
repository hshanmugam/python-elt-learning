# Topic 6 Exercises

1. Write a `try` block that converts a string to an integer.
2. Catch a `ValueError` if conversion fails.
3. Print a message in a `finally` block.
4. Import the `logging` module.
5. Configure logging with level `INFO`.
6. Log a message saying a pipeline started.
7. Log an error message for a rejected record.
8. Write logic that skips a bad amount but continues processing.
9. Explain when you should use `WARNING` instead of `ERROR`.
10. Catch a `KeyError` when a record is missing a required field.
11. Write a function that returns `None` on conversion failure.
12. Explain why logs are important for ETL support.
