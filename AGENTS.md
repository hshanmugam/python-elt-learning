# LearnPython Project Instructions

## Goal

Create a structured Python learning repository focused on helping me master Python for ELT and ETL pipelines.

## Learning Requirements

- Teach Python concepts that are directly useful for ELT and ETL work.
- Document the Python concepts and topics clearly.
- Create a documented learning plan and roadmap.
- Under each topic, provide detailed and easy-to-understand explanations.
- Include examples that relate to real ETL and ELT scenarios when possible.

## Topic Content Requirements

For each topic:

- Create separate files for the concept.
- Include commented Python code examples.
- Include at least 10 exercises.
- Include answers for the exercises.
- Include a short mini quiz.
- Keep the material beginner-friendly but practical.

## Project Requirements

- Create mini ETL and ELT style projects using the concepts taught.
- Make the projects reflect realistic company-style data engineering scenarios.
- Organize the projects so they build on the earlier learning topics.

## Repository Requirements

- Place all files in the local Git folder `C:\Projects\Etl-Elt\LearnPython`.
- Use a clean and logical folder structure.
- Organize the material in a clear learning sequence from fundamentals to advanced topics.

## Git and GitHub Requirements

- If the folder is not already a Git repository, initialize it.
- Stage and commit the work with clear commit messages.
- Push the completed work to the GitHub repository:
  `https://github.com/hshanmugam/python-elt-learning`

## Quality Standards

- Use clear file and folder names.
- Keep explanations elaborate, clear, and easy to follow.
- Use practical examples with business-style datasets and pipeline scenarios.
- Add comments in code that explain what the code is doing.
- Prefer structured, well-documented content over short or incomplete notes.

## Preferred Outcome

I want this repository to become a complete Python learning path for ETL and ELT, including:

- fundamentals
- intermediate topics
- advanced data engineering topics
- hands-on projects
- documentation for self-study and revision
